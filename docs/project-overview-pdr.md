# Project Overview & PDR — 2FA.media

## Purpose

Free, client-side TOTP code generator at **2fa.media**. Primary goal: attract organic search traffic for queries like "2fa code generator online" and funnel visitors to **goadsagency.com** via UTM-tagged CTAs.

## Funnel Goal

```
Search → 2fa.media → use free tool → see GOADS CTAs (4 touchpoints) → goadsagency.com
```

UTM touchpoints: `header` · `promo` · `footer` · `faq`
All links generated by `goadsUrl(content: GoadsContent)` in `src/data/site-links.ts`.

## Scope — MVP (shipped)

| In scope | Out of scope |
|---|---|
| Single-page SPA (EN only) | Multi-language / i18n |
| TOTP generation (RFC 6238, HMAC-SHA1, 30 s, 6-digit) | HOTP, SHA-256/512 variants |
| Multi-secret input (one per line or pipe-separated) | QR code scanning |
| Copy per-code + copy-all + export .txt | Account management / persistence |
| Countdown ring (30 s auto-refresh) | User accounts, saved secrets |
| FAQ section (8 items) | Server-side rendering of codes |
| SEO: OG image, sitemap, robots, JSON-LD | Paid features |
| Vercel deployment, zero env vars | External TOTP API calls |

## Key Decisions

### 1. No TOTP library
Hand-rolled RFC 6238 via `crypto.subtle` (Web Crypto API). Rationale: zero npm dep for crypto = auditable, smaller bundle, no supply-chain risk. Verified against RFC 6238 Appendix B vectors via `scripts/verify-totp.mjs`.

### 2. Client-side only — privacy by architecture
Secrets never reach a server. No localStorage, no sessionStorage, no cookies. Tab close = data gone. This is a trust signal stated explicitly in the FAQ and verifiable via DevTools Network tab.

### 3. Single page, no routing
One URL (`/`), one page. Keeps SEO simple (single canonical), deployment trivial, and codebase minimal.

### 4. Single `"use client"` boundary
Only `two-fa-tool.tsx` is a client component. All sections (hero, tool wrapper, promo, FAQ) are RSC. Minimises client bundle.

### 5. Data as single source of truth
`FAQ_ITEMS[]` in `src/data/faq-content.ts` drives both the rendered FAQ section and the `FAQPage` JSON-LD — no duplication.
`goadsUrl()` in `src/data/site-links.ts` is the only place UTM parameters are constructed.

### 6. Security headers at edge
`next.config.ts` sets `X-Content-Type-Options`, `Referrer-Policy: strict-origin-when-cross-origin`, `X-Frame-Options: DENY`, and `Permissions-Policy` on all routes. CSP deferred to post-launch (start in report-only mode).

## Acceptance Criteria (MVP)

- [x] Generates valid TOTP codes matching RFC 6238 test vectors
- [x] Auto-refreshes codes every 30 s at window boundary
- [x] Multi-secret input: one per line + pipe-separated rows
- [x] Copy single code, copy all, export .txt
- [x] No network requests during code generation (verifiable via DevTools)
- [x] Core Web Vitals green on Vercel (fully static, no SSR latency)
- [x] OG image, sitemap.xml, robots.txt present
- [x] JSON-LD: WebApplication + FAQPage schemas
- [x] All GOADS CTAs use `goadsUrl()` with correct `utm_content` per touchpoint

## Non-Functional Requirements

| Requirement | Target |
|---|---|
| Bundle size | No heavy crypto deps — `crypto.subtle` only |
| Availability | Vercel Edge — 99.99% SLA |
| Privacy | Zero secret persistence; verifiable client-only |
| SEO | Single canonical URL, complete meta tags, JSON-LD |
| Security | Security headers on all routes; CSP report-only post-launch |
| Maintainability | docs.maxLoc 800; design-system bundle in `docs/2fa-media-design-system/` |
